# 策略说明

本文档以当前代码实现为准，覆盖 `strategies/` 目录下 5 个策略的入场、出场、冷却和默认启用配置。

## 模块结构

```text
strategies/
  types.ts
  helpers.ts
  registry.ts
  s1.ts
  s2.ts
  s3.ts
  s4.ts
  s5.ts
```

统一接口由 `IStrategy` 定义，核心方法如下：

- `updateGuards(ctx)`: 每个 tick 更新冷却锁、峰值、偏差等守卫状态
- `checkEntry(ctx)`: 判断是否触发入场，返回 `up` / `down` 或 `null`
- `checkExit(ctx, direction)`: 判断是否触发止盈 / 止损 / 强平
- `onEntryFilled(ctx, direction)`: 买入成交后初始化持仓内状态
- `finalizeTick(diff)`: 将本 tick 的 diff 持久到策略内部，用于下一次穿越判断
- `resetState()`: 窗口切换时重置策略私有状态
- `getDescription()`: 返回前端悬浮说明
- `getStatePayload()`: 返回当前策略私有状态，供 `/api/state` 展示

## 后端状态机

所有策略共用同一套交易状态机，同一时刻只允许一个策略持仓：

```text
IDLE
  -> SCANNING
  -> BUYING
  -> WAIT_FILL
  -> HOLDING
  -> SELLING
  -> WAIT_SELL_FILL
  -> DONE / SCANNING
```

补充说明：

- `SCANNING`: 等待满足入场条件
- `WAIT_FILL`: 买单已发出，等待成交确认
- `HOLDING`: 已持仓，等待策略止盈或止损
- `WAIT_SELL_FILL`: 卖单已发出，等待成交确认
- 窗口切换时会整体重置策略运行态

## Strategy 1 / Regular+

源码：`strategies/s1.ts`

参数：

- 入场 diff: `35`
- 入场概率上限: `80%`
- 扫描窗口: `210s` 到 `50s`
- 中性区: `|diff| <= 25`
- 中性持续时间: `3000ms`
- 过热锁定: `diff >= 55` 且方向概率 `>= 85%`
- 追踪止损回撤: `20`
- 最低支撑位止损: `5`
- 最短持仓时间: `3000ms`
- 概率回撤止盈: 峰值 `>= 85%` 后回撤 `8%`
- 强制离场: 最后 `10s`

入场逻辑：

- `up`: `lastDiff < 35` 且当前 `diff > 35`，并且 `upPct < 80`
- `down`: `lastDiff > -35` 且当前 `diff < -35`，并且 `dnPct < 80`
- `upBlocked` 和 `downBlocked` 分方向独立锁定

出场逻辑：

- 阶梯止盈：从 `90%` 逐步提高到 `100%`
- 概率回撤止盈：峰值概率高于阈值后回撤即止盈
- diff 失守止损：`up` 持仓时 `diff <= 5`，`down` 持仓时 `diff >= -5`
- 追踪止损：有利 diff 相对峰值回撤 `20`
- 最后 `10s` 强平

## Strategy 2 / Regular

源码：`strategies/s2.ts`

参数：

- 入场 diff: `40`
- 入场概率上限: `78%`
- 扫描窗口: `220s` 到 `48s`
- 中性区: `|diff| <= 25`
- 中性持续时间: `3000ms`
- 过热锁定: `|diff| >= 55` 且方向概率 `>= 80%`
- diff 止损阈值: `5`
- 阶梯止盈起点概率: `92%`
- 阶梯止盈参与判断时间: 剩余 `12s` 以上
- 最后 `10s` 强制离场

入场逻辑：

- `up`: `lastDiff <= 40` 且当前 `diff > 40`，并且 `upPct < 78`
- `down`: `lastDiff >= -40` 且当前 `diff < -40`，并且 `dnPct < 78`
- 只在剩余 `220s` 到 `48s` 的时间窗内判定
- 需要最近 `8s` 内出现过一次中性区触碰，避免追在持续过热的单边上

冷却与封锁逻辑：

- 若 diff 尚未穿越入场阈值，但某方向概率已经过热，则标记 `upProbSeen` / `downProbSeen`
- 一旦出现 `upProbSeen` 或 `downProbSeen`，会同时锁定 `upBlocked` 与 `downBlocked`
- 若出现更强的过热状态：
  - `diff >= 55` 且 `upPct >= 80`
  - `diff <= -55` 且 `dnPct >= 80`
  也会同时锁定双向
- 双向锁只有在 `|diff| <= 25` 且持续 `3000ms` 后才会解除

出场逻辑：

- 阶梯止盈：从 `92%` 开始，随时间逐渐提高到 `100%`
- 最后 `10s` 按当前方向概率强制离场（`>= 50%` 记为止盈，否则记为止损）
- diff 固定止损：
  - `up` 持仓时，`diff <= 5`
  - `down` 持仓时，`diff >= -5`

实盘观察：

- `2026-04-15 11:07:33` 触发 `Strategy2` 入场，方向 `up`
- 下单金额 `1 USDC`
- 成交结果：
  - 实际成交份额约 `1.5873`
  - 成交价 `0.63`
  - 交易哈希 `0x31480f68610dfe6de4780bee3df7731381d4889b8b73a086ef0ce8892c3ce400`
- 日志中可见：
  - `entry triggered (1/1) dir=up amount=1`
  - `buy fill confirmed`

## Strategy 3 / Sweep

源码：`strategies/s3.ts`

参数：

- 扫描窗口: 最后 `60s`
- 入场 diff: `50`
- 概率上限: `95%`
- 止损 diff: `5`

入场逻辑：

- `up`: `diff > 50` 且 `upPct < 95`
- `down`: `diff < -50` 且 `dnPct < 95`

出场逻辑：

- 剩余 `40s` 以上：概率 `>= 98%` 止盈
- 剩余 `20s` 到 `40s`：概率 `>= 99%` 止盈
- 剩余 `10s` 到 `20s`：概率 `>= 100%` 止盈
- diff 固定止损：
  - `up` 持仓时，`diff <= 5`
  - `down` 持仓时，`diff >= -5`

## Strategy 4 / Reversal

源码：`strategies/s4.ts`

参数：

- 扫描窗口: 最后 `5s` 到 `1s`
- `up` 反转触发：`prevUpPct < 30` 且 `upPct > 60`
- `down` 反转触发：`prevUpPct > 70` 且 `upPct < 40`
- 止盈概率: `85%`
- 止损概率: `50%`

入场逻辑：

- 使用概率快速反转，而不是 diff 穿越

出场逻辑：

- 持仓方向概率 `>= 85%` 止盈
- 持仓方向概率 `<= 50%` 止损

## Strategy 5 / ProbChase

源码：`strategies/s5.ts`

参数：

- 扫描窗口: `240s` 到 `30s`
- 入场 diff: `25`
- 最小公平偏差: `10%`
- 止盈偏差收敛阈值: `3%`
- 止损偏差扩张倍数: `1.5x`
- 最大持仓时间: `30s`
- 最后 `8s` 强制离场

入场逻辑：

- `up`: `lastDiff <= 25` 且当前 `diff > 25`，同时 `fairProb - upPct >= 10`
- `down`: `lastDiff >= -25` 且当前 `diff < -25`，同时 `100 - fairProb - dnPct >= 10`

出场逻辑：

- 当前偏差 `<= 3%` 止盈
- 当前偏差扩张到入场偏差的 `1.5x` 止损
- 持仓超过 `30s` 强制离场
- 最后 `8s` 强制离场
- diff 反向穿越零轴时止损

## 运行时查看方法

可以通过以下接口查看当前策略状态：

- `/api/state`
- `/api/strategy/descriptions`

`/api/state` 中最关键的字段：

- `strategy.state`
- `strategy.activeStrategy`
- `strategy.direction`
- `strategy.roundEntryCount`
- `strategy.perStrategy.s2`
- `tradeHistory`
- `claimable`

截至本次文档更新时，默认运行状态是：

- 启用策略：仅 `S2`
- 当前运行态：`SCANNING`
- 当前无持仓

## 离线回放方法

本仓库已经内置策略回放脚本，可直接使用采集到的 `jsonl` 数据做离线复盘。

常用命令：

```bash
npm run replay:strategy -- --file backtest-data/2026-04-15.jsonl --strategy s2
```

同时回放全部策略并输出逐笔成交明细：

```bash
npm run replay:strategy -- --file backtest-data/2026-04-15.jsonl --strategy all --details
```

补充说明：

- `--strategy` 支持 `all`，也支持逗号分隔，例如 `s1,s2,s5`
- `--amount` 可覆盖模拟下单金额，默认值为 `1`
- 回放结果会输出：
  - `trades` 总成交笔数
  - `winRate` 胜率
  - `totalPnl` 总收益
  - `avgPnl` 单笔平均收益
  - `exitReasons` 各类出场原因统计
- 若 `openAtEnd > 0`，表示该数据样本在最后一个 tick 结束时仍有未正常出场的持仓
